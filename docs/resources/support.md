# Support

## Channels

| Channel       | Contact                                              | Response Time   |
|---------------|------------------------------------------------------|-----------------|
| Email         | [support@stendly.com](mailto:support@stendly.com)    | Within 24 hours |
| GitHub Issues | [github.com/stendly](https://github.com/stendly-dev) | Within 48 hours |

## Bug Reports

Open a GitHub issue with:

1. SDK version and language
2. Environment (Node.js/Python/.NET version)
3. Complete error message and stack trace
4. Minimal reproduction code
5. Steps to reproduce

## Feature Requests

Open a GitHub Discussion with:

1. Clear description of the feature
2. Use case and motivation
3. Any relevant examples or references

## Status Page

Check [status.stendly.com](https://status.stendly.com) for real-time API status and incident history.

## Security Issues

Report security vulnerabilities to [security@stendly.com](mailto:security@stendly.com).
